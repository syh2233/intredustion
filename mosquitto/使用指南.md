# Mosquitto MQTT Broker 使用指南

## 问题诊断

当前Mosquitto服务可能存在以下问题：
1. **监听配置问题** - 默认配置只监听localhost
2. **匿名连接限制** - 默认不允许匿名连接
3. **服务未正确启动**

## 解决方案

### 1. 启动Mosquitto服务

使用新的配置文件启动服务：

**PowerShell**
```powershell
# 在PowerShell中运行
restart_mosquitto.ps1
```

### 2. 验证服务状态

检查服务是否正常启动：

**PowerShell测试**
```powershell
cd E:\mos\mosquitto
.\test_connection.ps1
```

**CMD测试**
```cmd
cd E:\mos\mosquitto
test_connection.bat
```

**手动测试**
```cmd
cd E:\mos\mosquitto
mosquitto_sub.exe -h 192.168.24.23 -t test/topic -v
```

### 3. ESP32连接测试

ESP32代码已经配置为优先连接本地Mosquitto服务：
- 服务器地址：运行服务器机子的ip
- 端口：1883
- 无需用户名密码（允许匿名连接）

### 4. 监控ESP32数据

监控ESP32发送的数据：

```bash
# 监控所有传感器数据
mosquitto_sub.exe -h {ip} -t "esp32/fire_alarm/data" -v

# 监控报警消息
mosquitto_sub.exe -h {ip} -t "esp32/fire_alarm/alert" -v

# 监控所有ESP32消息
mosquitto_sub.exe -h {ip} -t "esp32/#" -v
```

## 配置文件说明

### mosquitto_simple.conf
```
listener 1883 0.0.0.0          # 监听所有网络接口
allow_anonymous true           # 允许匿名连接
connection_messages true       # 显示连接信息
log_timestamp true            # 显示时间戳
```

## 常见问题

### 1. 连接被拒绝
- 确保Mosquitto服务正在运行
- 检查防火墙是否阻止1883端口
- 确认使用正确的IP地址

### 2. 权限错误
- 确保配置文件允许匿名连接
- 检查用户权限设置

### 3. 端口占用
- 检查1883端口是否被其他程序占用
- 使用`netstat -an | findstr :1883`检查

## 服务管理

### 启动服务
```bash
mosquitto.exe -c mosquitto_simple.conf -v
```

### 停止服务
```bash
# 在命令行窗口按 Ctrl+C
```

### 安装为Windows服务
```bash
mosquitto.exe install -c mosquitto_simple.conf
net start mosquitto
```

## 测试命令

### 发布消息
```bash
mosquitto_pub.exe -h 192.168.24.23 -t test/topic -m "Hello MQTT"
```

### 订阅消息
```bash
mosquitto_sub.exe -h 192.168.24.23 -t test/topic -v
```

### ESP32数据监控
```bash
mosquitto_sub.exe -h 192.168.24.23 -t "esp32/fire_alarm/data" -v | jq
```

## 防火墙设置

如果需要从外部网络访问，请在Windows防火墙中添加例外：
1. 打开Windows防火墙设置
2. 添加入站规则
3. 允许TCP端口1883
4. 应用到所有网络类型

## 故障排除

### 1. 检查日志
启动时使用`-v`参数查看详细日志

### 2. 测试工具
使用提供的`test_connection.bat`测试连接

### 3. 网络诊断
```bash
ping {ip}
telnet {ip} 1883
```

按照以上步骤操作，应该能够解决MQTT连接问题。
